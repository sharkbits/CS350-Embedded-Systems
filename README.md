# CS350-Embedded-Systems

Throughout CS350 we were tasked with building a homemade thermostat that used a temperature and humidity sensor along with three buttons. Two buttons adjusted the target temperature and one cycled through system states. The project simulated a small embedded system that reads environmental data and responds to user input. We were given partially completed starter code that helped guide the structure while leaving key functionality for us to implement.
I felt I handled the technical aspects well, especially transitioning from the RPi.GPIO library to the gpiozero library. Since gpiozero is better maintained and easier to use, it made the code cleaner and more organized. I successfully integrated the sensor readings with the system logic and button controls. I also used AI tools like ChatGPT in VSCode as a code assistant to better understand my code, explore improvements, and get feedback on readability and best practices.
There are areas where I can improve, particularly documentation and version control. Documentation sometimes feels like an afterthought, and I would like more experience reviewing and working with other people’s code. To strengthen these areas, I plan to continue practicing with Git and GitHub, explore open source projects, and use AI tools thoughtfully to support my learning.
This project strengthened transferable skills such as hardware and software integration, state management, debugging, and writing organized, maintainable code. The structure and comments in the starter code provided a strong example of good practices, which influenced how I approached my implementation. As I enter my final term, I feel both overwhelmed and excited, but this project reinforced the importance of adaptability, maintainability, and continuous growth as I move forward in computer science.
